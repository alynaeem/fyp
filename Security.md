# Security Policy

## Reporting a Vulnerability

If you've discovered a security vulnerability in **DarkPulse**, please **do not** create a public GitHub issue.

We strongly encourage **responsible disclosure** to help protect users and infrastructure while we investigate and fix the issue.

### How to Report

Please report vulnerabilities **privately** using one of the following methods:

- **Security Advisory (Preferred)**  
  Submit a private security advisory through GitHub:
  - Go to the repository
  - Click **Security**
  - Click **Report a vulnerability**

- **Email**  
  Send detailed information to:  
  **📧 m.abd571870@gmail.com**

### What to Include

To help us triage and fix the issue faster, please include:

- A clear description of the vulnerability  
- Steps to reproduce (proof-of-concept if possible)  
- Affected components, endpoints, or modules  
- Potential impact and severity assessment  
- Any suggested mitigation (optional)

### Response Timeline

We aim to handle security reports with priority:

- **Acknowledgement:** within **8 hours**
- **Critical issues:** short-term mitigation or fix within **2 days**, whenever possible

### Scope

This policy applies to all components of DarkPulse, including but not limited to:

- API collectors
- Crawlers
- Security scanners (e.g., Trivy integration)
- Data processing pipelines
- UI / API interfaces
- Docker & deployment configurations

### Legal Safe Harbor

We will not pursue legal action against researchers who:

- Follow responsible disclosure practices  
- Avoid data destruction or service disruption  
- Do not exploit vulnerabilities beyond proof-of-concept  
- Act in good faith to improve the security of the project  

---

Thank you for helping keep **DarkPulse** and its users secure. 🔐
